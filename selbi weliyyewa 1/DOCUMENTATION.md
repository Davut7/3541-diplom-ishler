## 1. Что такое GAN?

### 1.1 Определение

**GAN (Generative Adversarial Network)** - Генеративно-состязательная сеть - это архитектура машинного обучения, состоящая из двух нейронных сетей, которые соревнуются друг с другом.

### 1.2 Компоненты GAN

```
┌─────────────────┐     ┌─────────────────────┐     ┌──────────────────┐
│   Случайный     │────▶│     ГЕНЕРАТОР       │────▶│  Сгенерированные │
│     шум (z)     │     │  (создает данные)   │     │     данные       │
└─────────────────┘     └─────────────────────┘     └────────┬─────────┘
                                                             │
                                                             ▼
┌─────────────────┐     ┌─────────────────────┐     ┌──────────────────┐
│   Реальные      │────▶│   ДИСКРИМИНАТОР     │────▶│   Real / Fake    │
│    данные       │     │ (различает данные)  │     │    (0 или 1)     │
└─────────────────┘     └─────────────────────┘     └──────────────────┘
```

**Генератор (Generator):**
- Принимает на вход случайный шум
- Создает "поддельные" данные (изображения, текст и т.д.)
- Цель: обмануть дискриминатор

**Дискриминатор (Discriminator):**
- Принимает реальные и сгенерированные данные
- Определяет, является ли вход реальным или поддельным
- Цель: правильно классифицировать данные

### 1.3 Процесс обучения

1. Генератор создает поддельные данные
2. Дискриминатор получает реальные и поддельные данные
3. Дискриминатор учится различать их
4. Генератор учится на ошибках дискриминатора
5. Процесс повторяется до достижения равновесия

### 1.4 Математическая формулировка

Минимаксная игра:
```
min_G max_D V(D,G) = E[log D(x)] + E[log(1 - D(G(z)))]
```

Где:
- G - генератор
- D - дискриминатор
- x - реальные данные
- z - случайный шум

## 4. Типы атак на GAN

### 4.1 Adversarial Examples (Состязательные примеры)

**Описание:** Небольшие возмущения входных данных, которые приводят к неправильной классификации.

**Методы:**
| Метод | Описание | Сложность |
|-------|----------|-----------|
| **FGSM** | Fast Gradient Sign Method - быстрая одношаговая атака | Низкая |
| **PGD** | Projected Gradient Descent - итеративная атака | Средняя |
| **C&W** | Carlini & Wagner - оптимизационная атака | Высокая |
| **DeepFool** | Минимальное возмущение для смены класса | Средняя |

**Формула FGSM:**
```
x_adv = x + ε * sign(∇_x J(θ, x, y))
```

**Уровень риска:** Высокий

---

### 4.2 Model Inversion (Инверсия модели)

**Описание:** Восстановление обучающих данных из выходов модели.

**Процесс атаки:**
1. Злоумышленник получает доступ к API модели
2. Отправляет множество запросов
3. Анализирует выходы и градиенты
4. Оптимизирует входы для восстановления данных

**Уровень риска:** Критический

**Почему опасно:** Может раскрыть конфиденциальные данные (медицинские снимки, личные фото)

---

### 4.3 Membership Inference (Атака членства)

**Описание:** Определение, был ли конкретный образец в обучающих данных.

**Методы:**
- Shadow Model Attack - создание теневых моделей
- Threshold-based Attack - анализ уверенности модели
- Label-only Attack - работа только с метками

**Уровень риска:** Средний

---

### 4.4 Data Poisoning (Отравление данных)

**Описание:** Внедрение вредоносных образцов в обучающие данные.

**Типы:**
| Тип | Описание |
|-----|----------|
| **Backdoor Attack** | Скрытый триггер активирует вредоносное поведение |
| **Clean-label Attack** | Атака без изменения меток |
| **Gradient-based** | Оптимизация ядовитых образцов |

**Уровень риска:** Критический

**Почему самая опасная:** Компрометирует модель на этапе обучения, влияет на все будущие выходы.

---

### 4.5 Mode Collapse Exploitation

**Описание:** Эксплуатация тенденции GAN генерировать ограниченное разнообразие.

**Как работает:**
1. GAN начинает генерировать однотипные выходы
2. Злоумышленник предсказывает поведение генератора
3. Использует предсказуемость для атаки

**Уровень риска:** Средний

---

### 4.6 Model Stealing (Кража модели)

**Описание:** Создание копии модели через API-запросы.

**Процесс:**
1. Отправка множества запросов к целевой модели
2. Сбор пар вход-выход
3. Обучение суррогатной модели
4. Дистилляция знаний

**Уровень риска:** Высокий

## 5. Механизмы защиты

### 5.1 Adversarial Training (Состязательное обучение)

**Описание:** Включение состязательных примеров в процесс обучения.

**Реализация:**
```python
# Псевдокод
for batch in training_data:
    # Генерация adversarial примеров
    x_adv = generate_adversarial(x, model)

    # Обучение на чистых и adversarial данных
    loss = loss_fn(model(x), y) + loss_fn(model(x_adv), y)
    loss.backward()
    optimizer.step()
```

**Эффективность:** 85%

### 5.3 Gradient Masking (Маскировка градиентов)

**Описание:** Скрытие или модификация градиентов для предотвращения градиентных атак.

**Методы:**
- Defensive distillation
- Gradient regularization
- Input transformation

**Эффективность:** 60%

**Ограничение:** Уязвим к transferability атакам

---

### 5.4 Differential Privacy (Дифференциальная приватность)

**Описание:** Добавление калиброванного шума для защиты приватности данных.

**Формула:**
```
Pr[M(D) ∈ S] ≤ e^ε × Pr[M(D') ∈ S] + δ
```

**Параметры:**
- ε (epsilon) - бюджет приватности
- δ (delta) - вероятность утечки

**Эффективность:** 90%

---

### 5.5 Ensemble Methods (Ансамблевые методы)

**Описание:** Использование нескольких моделей для повышения устойчивости.

**Типы:**
- Model averaging
- Boosting
- Bagging

**Эффективность:** 75%

## 9. Вопросы и ответы для защиты

### Вопрос 1: Что такое GAN и как он работает?

**Ответ:**
GAN - это генеративно-состязательная сеть, состоящая из двух нейронных сетей:
1. **Генератор** создает поддельные данные из случайного шума
2. **Дискриминатор** учится отличать реальные данные от поддельных

Они соревнуются в "игре" - генератор пытается обмануть дискриминатор, а дискриминатор пытается не быть обманутым. В результате генератор учится создавать очень реалистичные данные.

---

### Вопрос 2: Почему GAN уязвимы к атакам?

**Ответ:**
GAN уязвимы по нескольким причинам:

1. **Градиентная оптимизация** - атакующие могут использовать градиенты для создания adversarial примеров
2. **Запоминание данных** - GAN может запоминать обучающие данные, что делает возможным Model Inversion
3. **Двойная архитектура** - наличие двух сетей создает больше точек атаки
4. **Переобучение** - склонность к переобучению делает модель уязвимой

---

### Вопрос 3: Какая атака самая опасная и почему?

**Ответ:**
**Data Poisoning (Отравление данных)** - самая опасная атака, потому что:

1. Компрометирует модель на этапе обучения
2. Влияет на ВСЕ будущие выходы модели
3. Трудно обнаружить после обучения
4. Может включать скрытые бэкдоры

Например, злоумышленник может добавить изображения с определенным паттерном, и модель будет генерировать вредоносный контент при обнаружении этого паттерна.

---

### Вопрос 4: Как защитить GAN от атак?

**Ответ:**
Комбинация нескольких методов:

1. **Adversarial Training** (85% эффективность) - обучение на adversarial примерах
2. **Differential Privacy** (90% эффективность) - добавление шума для защиты приватности
3. **Input Preprocessing** (70% эффективность) - фильтрация входных данных
4. **Attack Detection** (80% эффективность) - обнаружение аномальных входов

Ни один метод не дает 100% защиты, поэтому важно комбинировать подходы.

### Вопрос 6: Как работает ваш симулятор атак?

**Ответ:**
Симулятор демонстрирует процесс атаки в несколько этапов:

1. **Выбор типа атаки** (FGSM, PGD, Model Inversion и т.д.)

6. **Отображение метрик** (perturbation, confidence, success rate)

Это помогает понять, как adversarial атаки влияют на выходы модели.
